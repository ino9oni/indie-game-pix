# Issue: ルートモードの中央寄せレイアウト対応

## 背景
- RouteMapの座標系が静的な `viewBox="0 0 800 400"` 前提のため、描画が画面左上に偏っている。
- 今後ノード分岐を追加する際に、座標を追加するだけで中央に収まる仕組みが必要。

## 要件
- ノード群の外接矩形を算出し、上下左右に48pxの余白を確保した `viewBox` を動的に適用する。
- `preserveAspectRatio="xMidYMid meet"` を指定し、画面中央にノード/エッジが並ぶようにする。
- Heroスプライトや吹き出し、ラベルなどの装飾は相対位置を保ち、既存の移動アニメーションを維持する。

## 完了条件
- ルートモード画面でノード群が中央に寄る。
- デバッグモードで任意ノードへ移動しても、線上に沿った移動が維持される。
- フォントサイズ100%/150%/200%のいずれでもレイアウトが崩れない。
