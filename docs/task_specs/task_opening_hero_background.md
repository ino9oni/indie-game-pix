# タスク仕様: オープニングと名前入力の背景統一

## 背景
- オープニング画面は現在 `assets/img/title.png` を利用しており、TASKLIST の指示である主人公の全身画像と一致していない。
- 名前入力モードは主人公全身画像を参照しているが、アセットパスが `/hero/hero_fullbody.png` であり、仕様のパス表記（`assets/img/character/hero_fullbody.png`）と差異がある。

## 目的
- オープニングと名前入力の両モードで同一の主人公全身画像を背景に固定表示し、仕様と実装の齟齬を解消する。

## 要件
1. オープニングモード（`screen === "opening"`）で、背景を `assets/img/character/hero_fullbody.png` に差し替える。
2. 名前入力モード（`screen === "name"` またはゲーム開始フローで NameEntry を表示している状態）でも同一のパスを参照する。
3. 背景画像は全面に表示しつつ、主要 UI（タイトル、ボタン、名前入力カード）の視認性を損なわないように既存のオーバーレイ調整を維持する。
4. GAMEDESIGN の該当セクションに背景画像変更を反映する。
5. 指定パスに画像ファイルが存在しない場合は、既存の `hero/hero_fullbody.png` をリネームまたは複製して整合性を取る。

## テスト観点
- オープニングと名前入力の両方で主人公全身画像が背景として表示される。
- 背景以外のモードの表示に影響が出ていない。
- GAMEDESIGN 記載と実装が一致する。
