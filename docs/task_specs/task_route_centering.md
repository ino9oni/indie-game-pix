# タスク仕様: ルートモードの中心寄せレイアウト

## 背景
- 現状のRouteMapコンポーネントは `viewBox="0 0 800 400"` の静的SVG座標を前提に座標を決め打ちしている。
- `src/game/route.js` のノード座標はX=120〜760, Y=80〜320付近に固定されており、ルート全体が画面左上寄りに表示される。
- 今後のルート分岐追加を見据えると、キャンバスサイズに応じて中央寄せ・スケーリングされる仕組みが必要。

## 目的
- ルートモード表示領域の中心にノード／エッジ群がバランスよく収まるようにする。
- 既存のヒーロー移動アニメーションやラベル配置との整合性を維持する。

## 要件
1. `RouteMap` は渡されたノード座標のバウンディングボックスを算出し、上下左右に十分な余白（例: 48px）を加えた `viewBox` を動的に設定する。
2. `route-svg` の `preserveAspectRatio` を `xMidYMid meet` とし、ビュー全体の中心にノード群が収まるようにする。
3. ノード・ラベル・Hero画像・エンカウント吹き出しは、計算した座標と余白で正しく描画され、従来通りの相対位置を保つ。
4. ウィンドウリサイズやフォントスケール変更時でも、中央寄せ状態が崩れないことを確認する。
5. 既存のクリック判定・移動制約・アニメーションタイミングは変更しない。

## 非機能要件
- 算出ロジックは `useMemo` 等でメモ化し、余計な再レンダリングを避ける。
- マージン値・最小幅高さなどは定数化し、今後の分岐追加時に調整しやすくする。

## テスト観点
- ルートモードを表示した際にノードが左右中央に寄っているかのスクリーンショット確認。
- デバッグモードで任意ノードにジャンプしてもヒーローが線上を移動することを確認。
- フォントサイズ100%/150%/200%いずれでも中央寄せが維持されることを目視確認。
