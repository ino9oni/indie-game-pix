# タスク仕様: スコア計算式の更新

## 背景
- 現行実装ではピクロスクリア時の加点式が「残り秒数 × 100 + クリアボーナス」。
- GAMEDESIGNの新要求に合わせ、秒数係数を1に変更し、スコアのインフレを抑える必要がある。
- スコア表示演出（スロットマシン風アニメーション／SE）は維持する。

## 目的
- ピクロスクリア時に計上されるスコアを「残時間（秒数） × 1 + クリアボーナス」へ変更する。
- 既存の演出体験（加算アニメーション、効果音、フェードなど）を損なわずに調整する。

## 要件
1. `GAMEDESIGN` のスコア計算式記述を新式に更新し、関連する記述（例: 係数100の説明、チップSE音量等）は現状維持で良いことを確認する。
2. `src/App.jsx` の `awardScore` ロジックを更新し、獲得スコアを `remainingSeconds * 1 + bonus` で算出する。
3. 演出用カウンター（アニメーション時間やイージング）は、必要に応じて値域の縮小に合わせて過度に短くならないように最小時間を保持する。
4. ゲーム内スコア保存（`score`, `displayScore`, localStorage）においても新式の数値が反映されること。
5. クリアボーナスの定義値（practice など）は変更しない。

## 非機能要件
- 変更は既存のテストや未実装のESLint導入に影響を与えないよう配慮する。
- 小数秒は切り捨てて整数計算する（現行仕様を踏襲）。

## テスト観点
- 各難易度でクリアした際、コンソールやUI表示で加算値が「残秒 + ボーナス」になっていることを確認。
- 残り0秒でクリアした場合、ボーナスのみが計上されること。
- 従来のセーブデータを利用してもスコア表示が破綻しないこと（ローカルストレージとの互換性）。
