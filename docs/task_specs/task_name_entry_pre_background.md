# タスク仕様: 名前入力画面前の背景差し替え

## 背景
- GAMESTART モード（名前入力前の導入会話）は現在ランダム背景に依存しており、タスク指示の `assets/img/character/hero/hero_fullbody.png` が適用されていない。

## 目的
- 名前入力に入る前の導入パートでも主人公の全身画像を背景として固定し、遷移前後で世界観を統一する。

## 要件
1. `screen === "gamestart"` かつ名前入力ウィンドウが表示されていないタイミングでも `assets/img/character/hero_fullbody.png` を背景に使用する。
2. 背景処理は既存の `Background` コンポーネント経由で行い、他モードの既存振る舞いを変えない。
3. 同仕様を `GAMEDESIGN` に追記し、GAMESTART モードの背景について明文化する。
4. 背景画像は全画面にフィットし、透過やオーバーレイ演出はこれまでと同じ条件を保つ。

## テスト観点
- New Game 後に表示される導入会話（名前入力前）で主人公全身画像が背景に固定表示される。
- 名前入力表示中・終了後の画面遷移に影響がない（既存の背景指定が崩れない）。
- 他モードの背景がランダム化されたままなど、既存の仕様へ副作用が無い。
