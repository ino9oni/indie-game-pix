# タスク仕様: Quit後のオープニング戻り挙動

## 背景
- 現行実装ではピクロス画面の `Quit` ボタン押下時にルート画面へ戻る挙動となっており、TASKLIST の「再度最初からプレイする場合はオープニングへ遷移」指示とズレがある。
- ゲームオーバーで `Quit` を選んだ後も、BGM が停止したままになるケースがある。

## 目的
- `Quit` 操作やゲームオーバーからの再開時に必ずオープニングモードへ戻し、オープニングBGMを再生状態にする。

## 要件
1. ピクロス画面右上の `Quit` ボタン押下時、バトル関連状態をリセットした上で `screen` を `"opening"` に変更する。
2. ゲームオーバー画面の `Quit` でも同様にオープニングへ遷移し、BGM を `TRACKS.opening` へ戻して再生する。
3. 遷移時には敵AIタイマーやスペル効果などバトル固有のリソースを停止・初期化する。
4. オープニングへ戻ってから `New Game` / `Continue` を選択できるよう、必要な最小限の進行状態（セーブデータ）は保持するが、当該バトルセッションの残状態は破棄する。
5. GAMEDESIGN に本挙動とBGM再生の仕様を追記する。

## テスト観点
- ピクロス画面で `Quit` を押すとオープニング画面が表示され、BGM が直ちに再生される。
- ゲームオーバー → `Quit` でも同じ挙動となる。
- オープニングから `New Game` / `Continue` を選べる状態が維持されている。
- バトル固有タイマーやスペル効果が遷移後に残存しない。
