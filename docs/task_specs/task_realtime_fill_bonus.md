# タスク仕様: ピクロスモードのリアルタイム正解ボーナス

## 背景
- 現行実装ではピクロスの正解判定は Submit 時のみで、途中経過で正しいマスを確定してもスコアに反映されない。
- GAMEDESIGN のタスクリストでは、正解マスを塗った瞬間に特別ボーナス (100 点) を加算するリアルタイム判定が求められている。

## 目的
- プレイヤーが正しいマスを塗ったタイミングで即時にボーナスを獲得できるようにし、プレイフィールを向上させる。
- 誤操作や塗り直し時にスコアが暴走しないよう、安全にボーナスを管理する。

## 要件
1. 正しいマス (solution[r][c] === true) を初めて Fill 状態にした瞬間に特別ボーナス 100 点をスコアへ加算する。
2. 同一マスでのボーナスは 1 回のみとし、塗り直しや取り消しで再度加算されないようにする。
3. Submit 時の通常スコア計算処理（残時間×1 + クリアボーナス）とリアルタイムボーナスが重複加算され、表示も連続アニメーションとして破綻しないよう保つ。
4. ボーナス加算時には既存のスコアカウント演出・効果音を利用し、他の演出（ピクロスクリックエフェクト等）と競合しない。
5. ボーナス用の常数 (100) はコード上で定数化しておき、将来の調整が容易なようにする。

## 非機能要件
- ローカルストレージに保存される総スコアがリアルタイム加算を含んだ値になるよう、既存のスコア同期ロジックと整合させる。
- 既存のデバッグモード挙動（任意ノードジャンプなど）へ影響を与えない。

## テスト観点
- 正解マスを塗ると 100 点ずつスコアが即時加算され、表示がアニメーションすることを確認。
- 同一マスを塗り直しても 1 回しか加算されないこと。
- Submit で全問正解した際、リアルタイムで既に加算された分が重複せず、最終スコアが期待通りになること。
- 残り時間 0 でもリアルタイムボーナスが加算されること。
