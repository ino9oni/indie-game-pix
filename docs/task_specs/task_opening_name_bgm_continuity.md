# タスク仕様: オープニング〜名前入力間のBGM継続

## 背景
- 現状のBGM切り替えロジックでは、オープニングからゲーム開始 (`gamestart`) や名前入力 (`name`) へ遷移した際に `TRACKS.route` 等へフェードしており、TASKLIST の「同じBGMを再生する」指示と一致していない。
- BGM が切り替わるとフェードアウトが発生し、音楽が途切れるように聞こえる。

## 目的
- オープニング〜名前入力フロー全体で同一BGM（オープニングテーマ）をシームレスに再生し続ける。

## 要件
1. `opening`・`gamestart`・`name` の各スクリーンでは `TRACKS.opening`（もしくは同値のトラック）を使用し、遷移時にフェードアウトや停止を行わない。
2. 直前と同じトラックを再生する場合は、`bgm.play` が連続再生を維持するため再初期化を避ける（既存ロジックの再利用で可）。
3. BGM継続に伴い、GAMEDESIGN の該当セクションへ仕様追記を行う。
4. 他モード（プロローグ、ルート、バトル等）のBGM切り替え挙動に影響を与えない。

## テスト観点
- オープニング → ゲーム開始 → 名前入力の遷移中にBGMが途切れず鳴り続ける。
- `Pause` や他モードに移った際は従来通りのBGMに切り替わる。
- GAMEDESIGN 記載と実装が一致する。
