# タスク仕様: 対戦モードのレイアウト調整

## 背景
- TASKLIST で対戦モードの各コンポーネントの表示位置が細かく指示されている。
- 現状の UI ではスコアやスペル表示が左右共通の領域に集約されており、指定の位置に配置されていない。

## 目的
- プレイヤー側（画面左）と敵側（画面右）の各コンポーネントを所定の位置に再配置し、視覚的な左右対称と役割の分離を実現する。

## 要件
1. HERO（画面左）パネル
   - 左上: 小さなキャラクター表情（アバター）を表示する。
   - 中央上: スペル表示領域を確保し、発動中スペルを表示／未発動時はプレースホルダーを表示する。
   - 右上: スコア（勝利数）を表示する。
   - 中央: ピクロスボードを中央寄せで表示する。
2. ENEMY（画面右）パネル
   - 左上: 敵キャラクターの表情（アバター）を小さく表示する。
   - 中央上: スペル表示領域（敵の発動スペルを表示）を配置する。
   - 右上: 敵のスコアを表示する。
   - 中央: 敵ピクロスボードを配置する。
3. 進捗バーは各陣営のボード下部に配置し、レイアウト崩れを防ぐ。
4. 既存のスペルエフェクト（ログ・発射演出）は維持しつつ、新しい位置に合わせた UI に更新する。
5. 小画面（<=960px）では縦並びに再配置するが、アバター → スペル → スコア → ボード → 進捗 の順序を維持する。
6. GAMEDESIGN を更新して新レイアウト方針を明文化し、TASKLIST の該当項目を完了状態にする。

## テスト観点
- プレイヤー側のアバター・スペル・スコアが左パネル上部に正しい順序で表示される。
- 敵側も同様に指定位置へ表示される（左右対称）。
- スペル発動時に各パネルのスペル表示領域がハイライトされる。
- ウィンドウ幅を狭めた際も縦方向に順序を保ち、情報が重ならない。
