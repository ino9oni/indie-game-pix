# タスク仕様: コイン計上時のSE演出変更

## 背景
- 現在はスコア加算中に `score_chiptune` をループ再生しており、加算が終わるまで鳴り続ける。
- TASKLIST の改善項目では、コイン音を短いチャイム1回に絞り、より軽快なフィードバックにすることが求められている。

## 目的
- スコア加算時のサウンドを1回限りの高音チャイムへ変更し、冗長に鳴り続けないようにする。

## 要件
1. `AudioManager` のスコア加算用メソッドを変更し、サウンドをループ再生しない。
2. 既存の `score_chiptune.wav` を利用し、短く高音のチャイム感が出るよう再生ゲインと速度を調整する。
3. 再生は残時間スコアやリアルタイムボーナス加算のトリガーと連動し、複数回連続で鳴っても破綻しないようにする。
4. 音量は従来同様、BGM を覆わない程度に抑える。
5. GAMEDESIGN のスコア加算演出仕様にも変更点を追記する。

## 非機能要件
- 変更後も既存のスコア演出アニメーションやフェード処理との整合を保つ。

## テスト観点
- スコア加算時にチャイムが1回だけ鳴ることを確認する。
- 連続でスコア加算しても重複再生が問題なく処理されること。
- スコア加算の終了時に不要なフェードアウト処理が残っていないこと。
