# タスク仕様: ピクロス盤面枠の横幅調整

## 背景
- 現状のピクロスモードでは `.board` / `.enemy-board` が `1fr` を含むグリッド定義のため、盤面枠が列幅いっぱいに広がり左側が不自然に伸びて見える。
- 仕様ではピクロス盤面を中央に配置し、盤面サイズに応じて枠がフィットすることを求めている。

## 目的
- ピクロス枠を盤面の実寸に合わせ、左右に余計な余白が伸びないようにする。
- ヒント列との整合性を保ちながら、プレイヤー盤と敵盤のレイアウトを崩さず配置する。

## 要件
1. `GameBoard` と `EnemyBoard` が出力する盤面枠を、ピクセルサイズに合わせて自動幅で表示する。
2. `.board` および `.enemy-board` のグリッド定義から `1fr` の列を取り除き、ヒント列とグリッド列だけが必要幅で並ぶようにする。
3. `.board-frame` が余白で引き伸ばされないよう、コンテンツ幅に合わせて中央寄せする（必要に応じて `max-content` や `inline-flex` を活用）。
4. 変更は敵味方双方の盤面に適用し、レスポンシブ時の表示崩れがないことを確認する。
5. GAMEDESIGN にピクロス枠が盤面サイズに追従する旨を追記する。

## テスト観点
- プレイヤー盤・敵盤ともに枠の左端が画面中央まで伸びず、盤面幅に合わせて収まっている。
- ウィンドウサイズを変えても盤面が中央に保たれ、透明度や装飾は維持される。
- 他モードのレイアウトへ影響が出ていない。
